# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability.

Use GitHub's private vulnerability reporting if it is enabled for the repository,
or contact the maintainer directly through the GitHub profile.

Include:

- A clear description of the issue.
- Steps to reproduce.
- The affected version or commit.
- Any relevant logs or screenshots with private data removed.

## Privacy Expectations

Codex Context Notes stores notes locally and does not upload them to a backend.
Security-sensitive changes should preserve that local-first model unless there
is an explicit design discussion and review.
